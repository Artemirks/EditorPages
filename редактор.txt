Цели. Конструктор лендингов:

Вставка блоков по нажатию на блок с плюсом. После нажатия на плюс, становится активной кнопка создать рядом с выбором типа.

Режим редактирования блока при нажатии на него. Блок выделяется рамкой.

Каждый объект обладает основными свойствами: удаление блока, перемещение блока - две кнопки, переместить вверх, переместить вниз (только для основных элементов, у которых родитель page). Нужно отслеживать, что если блок первый, то выше его переместить нельзя, аналогично с последним блоком. 
Изменить - сохранение свойств. При нажатии пробегает по элементам массива props, считывает свойство value каждого свойства и присваивает значение соответсвующему свойству блока. Также пробегается по элементам массива childs и добавляет соответвующий html код на страницу. 

Режим редактирования - свойства редактирования появляются в поле под основными кнопками (создание блока, выбор типа, удаление блока, перемещение блока вверх-вниз)

При нажатии на плюс на его место вставляется блок с выбранным типом и под ним распологается блок с плюсом

Каждый блок представляет из себя объект. Свойства объекта: obj - ссылка на соответствующий DOM-объект. childs - массив, который содержит дочерние элементы. parent - ссылка на родительский объект, props - массив, который включает в себя объекты свойств, isParent - булевое свойство, показывающее, может ли элемент иметь дочерние элементы.

Объект свойств. Значение - все, что можно сделать с блоком. Свойство: value - значение данного свойства, name - название свойства, propose - назначение: style - вставляется в стили, action - действие, ничего не делать. Методы: makeForm - отвечает за отрисовку кнопки или поля редактирования данного свойства

Типы блоков: Меню, шапка, блок с текстом, картинка, блок с картинкой и текстом, видео, подвал, сетка

Несколько разновидностей каждого блока (различные представления меню, шапки, блоков с текстом)

Добавление элементов в блоки также является свойством блока

Возможность сохранения HTML-кода, получившийся страницы, для последующего использования и сохранения структуры страницы, для последующей редакции. 

Создание элементов редактора идет через js. Изначально на страницы только div обертка для редактора. Создается два поля: для вставки элементов и их отображения и поле редактирования. Изначально на нем неактивная кнопка создать и выбор типа. 

Объект элемента с плюсом хранит ссылку на элемент, который распологается сверху и который распологается снизу.

Есть базовый тип page, который является родителем для всех остальных типов

Есть тип new, который является начальной оберткой для остальных типов, у него есть свойства выбора типа. Действие у него замена свойств, на свойства типа

Алгоритм:

1. Страница создается, изначально в поле отображения только блок с плюсом для добавления нового блока. 
2. При нажатии на блок с плюсом. Кнопка создать становится активной.
3. При нажатии на кнопку создать добавляется блок с выбранным типом на место блока с плюсом. Блок с плюсом распалогается ниже вставленного блока.
4. При нажатии на блок вокруг него образуется красная рамка. В поле редактирования под кнопками создать и типа создаются кнопки удалить, переместить вверх и переместить вниз. После пробегаемся по массиву childs (если элемент может иметь дочерние элементы) и добавляем их в специальное поле, где мы можем менять их положение внутри родителя и добавить новый элемент. При добавлении элемента появляется форма с выбором типа, нужно в родительском элементе указывать какого типа могут быть дети. После пробегаемся по массиву props и создаем поля для ввода данных. После всего этого кнопка изменить
5. При нажатии на кнопку изменить сначала пробегаемся по массиву childs и рисуем детей (если элемент может иметь дочерние элементы), потом пробегаемся по полю props и записываем необходимые свойства с стили, другие в места преднаначенные для них, с оставшимеся ничего не делаем. После вставляем получившеюся строку в свойство объекта и вставляем на страницу за место старого блока. 
6. Перемещение блоков. Вниз - находим ближайщий к блоку нижний плюс, если у того плюса есть нижний элемент меняем их местами. Также необходимо у верхнего плюса поменять элемент. Если у ближайшего нижнего плюса нет нижнего элемента, то ничего не делаем. Вверх - все так же, только нижний плюс меняем на верхний.
7. Также должна быть кнопка сохранить, это основная кнопка. При ее нажатии формируется два файла. Один со структурой классов - json. Нужен для редактирования и востановления страницы в редакторе. Второй файл с html для использования
